## 方向生命周期管理

### 状态流转

```
提出 → 激活 → 执行 → 完成 → 归档
         ↓
       废弃 (如果方向不可行或优先级降低)
```

### 各状态定义

| 状态 | 含义 | 操作 |
|------|------|------|
| **提出** | 新方向建议，尚未评估 | 评估可行性，分配优先级 |
| **激活** | 已确认可行，进入活跃池 | 更新到活跃方向池，等待执行 |
| **执行** | 正在研究中 | 标记为当前研究方向 |
| **完成** | 研究结束，有明确产出 | 移至归档区，释放活跃池位置 |
| **废弃** | 方向不可行或优先级降低 | 记录原因，不再追踪 |
| **归档** | 历史记录，供参考 | 不再参与活跃轮询 |

## 更新规则

### 1. 方向提出阶段
当研究产生新方向建议时：
- **如果活跃池有空位**：直接添加，标记为"提出"
- **如果活跃池已满**：
  - 对比新方向与现有方向的优先级
  - 如果新方向优先级更高，替换最低优先级的现有方向
  - 被替换的方向标记为"暂缓"（可恢复）

### 2. 方向执行阶段
每个研究周期开始前：
1. 检查活跃池中的方向状态
2. 选择**优先级最高且状态为"待执行"**的方向
3. 如果没有待执行方向，执行基础方向轮询
4. 执行中的方向更新状态为"执行中"

### 3. 方向完成阶段
研究完成后：
- 更新方向状态为"完成"
- 添加研究成果摘要链接
- 下个周期自动移至归档区
- 释放活跃池位置

### 4. 方向评估标准

**优先级评估**:
- 🔴 **高**: 可能产生突破性进展，或有阻塞其他方向的风险
- 🟡 **中**: 重要但非紧急，可以并行推进
- 🟢 **低**: 锦上添花，有空闲资源时再执行
- ⚪ **待定**: 需要更多信息才能评估

**可行性评估**:
- ✅ 有明确的输入输出定义
- ✅ 有可参考的现有工作
- ✅ 在1-5个研究周期内可完成
- ❌ 过于抽象，无法落地
- ❌ 依赖尚未完成的其他方向
