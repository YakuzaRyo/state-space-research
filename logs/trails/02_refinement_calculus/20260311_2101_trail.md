#### 搜索主题3: Morgan精化演算基础
**关键词**: "program refinement calculus Morgan laws weakest precondition formal verification"

**核心发现**:

**Morgan精化演算基础**:
- 规范语句: `w:[pre, post]`
  * w: 可变变量框架(frame)
  * pre: 前置条件
  * post: 后置条件(含初始值x₀)

- 核心精化法则:
  * **Skip Law**: 若 `pre ⇒ post`，则 `w:[pre, post] ⊑ skip`
  * **Assignment Law**: 若 `pre ⇒ post[E/x]`，则 `w,x:[pre, post] ⊑ x := E`
  * **Sequential Composition**: `w:[pre, post] ⊑ w:[pre, mid]; w:[mid, post]`
  * **Strengthen Postcondition**: 若 `post' ⇒ post`，则 `w:[pre, post] ⊑ w:[pre, post']`
  * **Weaken Precondition**: 若 `pre ⇒ pre'`，则 `w:[pre, post] ⊑ w:[pre', post]`

- Weakest Precondition演算:
  * `wp(skip, Q) = Q`
  * `wp(x := E, Q) = Q[E/x]`
  * `wp(C1; C2, Q) = wp(C1, wp(C2, Q))`
  * `wp(if B then C1 else C2, Q) = (B ∧ wp(C1, Q)) ∨ (¬B ∧ wp(C2, Q))`

### Step 2: 提出假设 (3-5分钟)

#### 核心假设

**H1: 程序精化演算可作为LLM代码生成的语义约束框架**
- 规范语句 `w:[pre, post]` 定义了代码必须满足的行为契约
- 精化法则限制了从规范到代码的合法转换路径
- LLM只能在预定义的精化法则空间内选择，而非自由生成

**H2: Weakest Precondition演算可实现系统化验证**
- 对LLM生成的代码，可计算其weakest precondition
- 验证 `pre ⇒ wp(code, post)` 是否成立
- 验证失败时可通过反例反馈引导LLM修正

**H3: 语法约束与语义约束可形成双重保障**
- XGrammar等工具保证代码"形式正确"(符合语法)
- 精化演算保证代码"行为正确"(满足规范)
- 两者结合可实现从语法到语义的端到端约束

**H4: 精化法则可作为LLM的结构化生成约束**
- 类似于XGrammar使用CFG约束token生成
- 精化法则可约束代码变换的合法序列
- 可将精化选择建模为结构化预测问题

### Step 5: 调整方向计划 (2-3分钟)

#### 研究发现总结

**核心洞察**:
程序精化演算通过以下机制约束LLM生成:

1. **状态空间限制**: 规范语句 `w:[pre, post]` 定义了有效程序行为空间
2. **转移限制**: 仅允许预定义的精化法则(如Assignment、Sequential Composition)
3. **验证闭环**: 每步生成证明义务，ATP验证通过后方可继续
4. **反馈循环**: 验证失败提供反例，引导LLM修正

**与XGrammar的协同**:
```
+-------------------+     +-------------------+     +-------------------+
|   Specification   | --> |  LLM Generator    | --> |  Generated Code   |
|   w:[pre, post]   |     |  (constrained)    |     |                   |
+-------------------+     +-------------------+     +-------------------+
         |                      |                           |
         v                      v                           v
+-------------------+  +-------------------+      +-------------------+
| Refinement Laws   |  | XGrammar (Syntax) |      |  Verification     |
| (Semantic Guide)  |  | (CFG Constraints) |      |  (wp + SMT)       |
+-------------------+  +-------------------+      +-------------------+
```
