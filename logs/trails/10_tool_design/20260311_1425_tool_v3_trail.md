     - 状态转换通过返回不同泛型参数的Builder实现
     - 零运行时开销：PhantomData是ZST

## 关键发现

### 1. Typestate Builder模式核心机制

```rust
// Marker types
struct Unset;
struct Set;

// Builder with type parameters tracking state
pub struct DatabaseConfigBuilder<H, P, U, PW, D> {
    // ...
    _phantom: PhantomData<(H, P, U, PW, D)>,
}

// Only fully configured builder can build
impl DatabaseConfigBuilder<Set, Set, Set, Set, Set> {
    pub fn build(self) -> DatabaseConfig {
        // No runtime checks needed!
    }
}
```

### 2. 可选字段设计模式

可选字段不追踪状态，减少类型复杂度：

```rust
impl<H, P, U, PW, D> DatabaseConfigBuilder<H, P, U, PW, D> {
    pub fn timeout_ms(mut self, timeout: u64) -> Self {
        self.timeout_ms = timeout;
        self  // Same type returned
    }
}
```

### 3. CLI命令选择类型安全

```rust
pub struct NoCommand;
pub struct HasCommand;

impl CliConfigBuilder<NoCommand> {
    pub fn build_command(...) -> CliConfigBuilder<HasCommand> { ... }
    // No build() method here!
}

impl CliConfigBuilder<HasCommand> {
    pub fn build(self) -> CliConfig { ... }
}
```
