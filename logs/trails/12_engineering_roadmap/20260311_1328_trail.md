## Step 2: 提出假设 (3-5分钟)

### 核心假设

**H1: 六层渐进式边界架构**
```
感知层 → 状态层 → 决策层 → 执行层 → 反馈层 → 治理层
  L1       L2       L3       L4       L5       L6
```
- 每层独立状态机
- 层间通过明确契约通信
- 可独立开发、测试、部署

**H2: 渐进式部署策略**
- 从L1开始逐步添加层
- 每层验证后再进入下一层
- 降低整体系统风险

**H3: Actor+状态机混合模式**
- Actor处理I/O边界（异步、网络）
- 状态机处理核心逻辑（纯函数、确定性）
- 与Polar Signals DST模式一致

**H4: Rust类型系统保证**
- 编译期验证状态转换合法性
- 零成本抽象
- 生产级性能

**H5: 闭环反馈机制**
- 执行结果通过反馈层生成事件
- 事件回到状态层更新状态
- 形成完整控制闭环

#### 六层协调器

```rust
pub struct ProgressiveAgent<S, P, A, H>
where
    S: StateSpace,
    P: DecisionPolicy<S::State>,
    A: Executable,
    H: FeedbackHandler<A::Output>,
{
    layer1_perception: PerceptionLayer,
    layer2_state: StateLayer<S>,
    layer3_decision: DecisionLayer<S::State, P>,
    layer4_execution: ExecutionLayer,
    layer5_feedback: FeedbackLayer<A::Output, H>,
    layer6_governance: GovernanceLayer,
}

pub async fn step(&mut self, input: RawInput) -> Result<AgentOutput, AgentError> {
    // L6: 治理检查
    self.layer6_governance.check_request()?;

    // L1: 感知
    let event = self.layer1_perception.process(input)?;

    // L2: 状态
    let snapshot = self.layer2_state.current_snapshot();

    // L3: 决策
    let plan = self.layer3_decision.decide(&snapshot.state, None);

    // L4: 执行
    let results = self.execute_plan(&plan).await?;

    // L5: 反馈
    let feedback = self.layer5_feedback.process(results);

    // 闭环: 反馈事件应用到状态层
    for event in feedback.events {
        self.layer2_state.apply_event(event)?;
    }

    Ok(AgentOutput { ... })
}
```

#### 六层渐进式边界的价值

| 层级 | 职责 | 状态机状态 | 输出契约 |
|------|------|------------|----------|
| L1 感知 | 输入处理 | Idle/Processing/Error | PerceivedEvent |
| L2 状态 | 状态管理 | Ready/Updating/Persisting | StateSnapshot |
| L3 决策 | 行动规划 | Idle/Evaluating/Decided | ActionPlan |
| L4 执行 | 外部交互 | Ready/Executing/Completed | ExecutionResult |
| L5 反馈 | 结果处理 | Waiting/Processing | FeedbackBatch |
| L6 治理 | 监控保护 | Healthy/Throttled/CircuitOpen | GovernanceDecision |

#### 渐进式部署策略

```
Phase 1: L1 + L2 (MVP)
- 感知层 + 状态层
- 可处理输入并维护状态
- 无外部依赖，易于测试

Phase 2: + L3 (决策能力)
- 添加决策层
- 基于状态生成行动计划
- 可模拟执行

Phase 3: + L4 (执行能力)
- 添加执行层
- 与外部系统交互
- 需要错误处理/重试

Phase 4: + L5 (闭环)
- 添加反馈层
- 执行结果影响状态
- 形成完整循环

Phase 5: + L6 (生产就绪)
- 添加治理层
- 限流/熔断/监控
- 生产级可靠性
```
