### 关键发现

#### 1. LLM-A* (EMNLP 2024)
- **论文**: "LLM-A*: Large Language Model Enhanced Incremental Heuristic Search on Path Planning"
- **核心贡献**: 使用LLM生成waypoints指导A*搜索
- **启发式修改**: `h_new(n) = h(n) + h_LLM(n)`
- **性能**: 44-57%操作减少，64-78%存储减少
- **关键洞察**: 牺牲admissibility换取效率

#### 2. LATS (ICML 2024)
- **论文**: "Language Agent Tree Search Unifies Reasoning, Acting, and Planning"
- **核心贡献**: 首个统一推理、行动、规划的MCTS框架
- **性能**: HumanEval 92.7% pass@1，WebShop 75.9
- **关键洞察**: 环境外部反馈显著优于纯自我批评

#### 3. RethinkMCTS (EMNLP 2025)
- **论文**: "RethinkMCTS: Refining Erroneous Thoughts in Monte Carlo Tree Search for Code Generation"
- **核心贡献**: 在代码生成前搜索thoughts，直接修正错误thoughts
- **创新**: Block-level执行反馈 + 细粒度口头反馈
- **关键洞察**: "正确推理过程导致正确代码"

#### 4. Tree of Thoughts (NeurIPS 2023)
- **论文**: "Tree of Thoughts: Deliberate Problem Solving with Large Language Models"
- **核心贡献**: 将LLM从token级决策提升到"thought"级决策
- **性能**: Game of 24成功率4%→74%
- **关键洞察**: BFS/DFS搜索思维树，支持回溯和前瞻

#### 5. AlphaCode/AlphaCode 2 (DeepMind)
- **AlphaCode**: 大规模采样+过滤+聚类
- **AlphaCode 2**: 增强beam search + 奖励模型 + 迭代修正
- **关键洞察**: 多样性促进+大规模采样可以替代复杂搜索

#### 6. A-CEoH (2025)
- **创新**: 将A*算法代码结构嵌入prompt
- **方法**: Algorithmic Prompt-Augmentation

#### 7. MCTS-AHD (ICLR 2025)
- **创新**: 首次将MCTS应用于LLM自动启发式设计
- **技术**: Progressive widening

### 技术方案对比

| 维度 | LLM-A* | LATS | RethinkMCTS | ToT |
|------|--------|------|-------------|-----|
| 搜索空间 | 路径规划 | 推理+行动+规划 | 代码生成 | 通用推理 |
| 核心算法 | A* + LLM启发式 | MCTS | MCTS + Rethink | BFS/DFS/Beam |
| 反馈来源 | LLM评估 | 环境反馈 | 执行反馈 | 自我评估 |
| 关键创新 | Waypoint引导 | 统一框架 | 错误thought修正 | Thought级决策 |

## Step 2: 提出假设 (10:32:45 - 10:34:00, ~1分钟)

基于Web Research发现，提出以下假设：

```
H1: LLM在状态空间搜索中更适合作为"相对排序器"而非"绝对评估器"
    - 置信度: 高
    - 理由: Kendall's Tau > 0.7研究表明LLM更擅长相对比较

H2: 分层搜索架构（粗粒度+细粒度）比单层搜索更适合复杂状态空间
    - 置信度: 高
    - 理由: L2 Pattern层粗粒度选择 + L3 Domain层细粒度搜索

H3: 外部验证反馈（执行/测试）比纯LLM自我评估更可靠
    - 置信度: 高
    - 理由: LATS使用环境反馈达到92.7% vs 自我批评方法

H4: 错误thought的"就地修正"比"重新采样"更高效
    - 置信度: 中
    - 理由: RethinkMCTS通过修正错误thought而非丢弃

H5: 自适应束宽根据LLM置信度动态调整可提升20-30%效率
    - 置信度: 中
    - 理由: 高置信度时减小width，低置信度时增大width

H6: LLM启发式在"结构化状态空间"（如类型约束空间）中更有效
    - 置信度: 中
    - 理由: 类型系统提供天然的状态边界和转换规则
```

    fn rank_states(&self, states: &[S]) -> Vec<(usize, f64)>;  // 相对排序

}

// Part 2: H1验证 - 相对排序 vs 绝对评估
pub fn h1_validate_ranking_vs_absolute()

// Part 3: H2验证 - 分层搜索架构
pub struct HierarchicalSearcher { pattern_heuristic, impl_heuristic }
// L2 Pattern层粗粒度选择 + L3 Domain层细粒度搜索

// Part 4: H3验证 - 外部反馈集成
pub trait Testable { fn run_tests(&self) -> TestResult; }

## 关键洞察总结

1. **相对排序优于绝对评估**: Kendall's Tau > 0.7表明LLM更擅长相对比较
2. **外部反馈至关重要**: LATS的92.7% vs 自我批评基线证明环境反馈的价值
3. **就地修正更高效**: RethinkMCTS策略比重新采样保留更多上下文
4. **分层架构适合复杂空间**: L2 Pattern + L3 Domain分层有效管理复杂度
5. **类型约束提供边界**: 结构化状态空间(类型系统)提升LLM导航效率
