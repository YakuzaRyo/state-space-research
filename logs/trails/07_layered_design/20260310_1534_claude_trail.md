#### 发现1: Rust-Analyzer的三层架构设计
来源: https://rust-analyzer.github.io//blog/2020/07/20/three-architectures-for-responsive-ide.html

**核心洞察**:
- **Map-Reduce范式**: 将分析分为简单的索引阶段和完整的分析阶段
- **索引阶段**: 每文件独立运行，解析并输出顶层声明（stub），类型以未解析字符串形式存储
- **完整分析**: 利用索引信息惰性地解析类型，结果memoized缓存
- **增量更新**: 索引可增量更新，缓存完全失效后重建，实现O(变更文件数)而非O(总文件数)

**对分层设计的启示**: Syntax层（索引）与Semantic层（解析）分离，通过延迟求值连接

#### 发现2: RCL类型检查器的实现模式
来源: https://ruudvanasseldonk.com/2024/implementing-a-typechecker-for-rcl-in-rust

**核心洞察**:
- **check_expr方法**: 融合类型推断和类型检查的核心工作马
- **期望类型传播**: 自顶向下传递expected type，实现精准错误定位
- **SourcedType**: 类型+来源追踪，形成格(lattice)结构支持join操作
- **TypeDiff三态**: Ok(静态通过) | Defer(运行时检查) | Error(静态错误)
- **单遍检查**: 不累积约束，边遍历边检查，保持快速和错误可解释性

**对分层设计的启示**: Semantic层到Pattern层可通过"类型过滤器"实现，类型检查即过滤器

#### 发现3: Parser Combinator与分层转换
来源: https://bodil.lol/parser-combinators/

**核心洞察**:
- **组合子基础**: map, pred, either, pair, left, right, one_or_more, zero_or_more等原语
- **解析即转换**: Parser<A>通过map转换为Parser<B>，对应Syntax→Semantic转换
- **递归下降**: 通过box递归处理嵌套结构，自然形成AST层级
- **错误恢复**: 通过either实现备选解析路径，支持丰富的语法变体

**对分层设计的启示**: 解析器组合子天然支持Syntax→Semantic层转换，map操作即语义提取

#### 发现4: 编译器分层架构通用模式
来源: 多篇论文和文档

**核心洞察**:
- **六阶段经典模型**: Lexer → Parser → Semantic Analysis → IR Gen → Optimizer → Code Gen
- **Syntax-Directed Translation**: 语法分析时嵌入语义动作，边解析边构建语义结构
- **Visitor模式**: 将遍历与操作分离，支持多种分析（类型检查、优化、代码生成）复用同一AST
- **Pipe-Filter架构**: 每个阶段是filter，通过pipe连接，输出成为下一阶段的输入

**对分层设计的启示**: 四层转换(Syntax→Semantic→Pattern→Domain)可映射为Pipe-Filter链

### Step 2: 提出假设 (3-5分钟)

**H1: 解析器组合子可实现语法层到语义层转换**
- 解析器组合子的map操作将原始token流转换为AST节点
- 每个组合子封装一种语法结构到语义结构的映射
- 通过类型系统保证转换的完整性

**H2: 类型检查可作为语义层到模式层的过滤器**
- 类型检查器遍历AST，过滤掉类型不匹配的节点
- 通过期望类型传播，将高层语义约束下推到低层实现
- 类型正确的AST子集形成"模式"层

**H3: 代码模板可实现模式层到领域层生成**
- 每种模式对应一个代码生成模板
- 模板参数由模式层的类型信息填充
- 生成目标领域特定代码（如Rust、SQL、配置等）

#### Layer 2: Semantic层 - Parser Combinator实现
```rust
pub trait Parser<T>: Clone {
    fn parse(&self, tokens: &[Token]) -> ParseResult<T>;
    fn map<F, U>(self, f: F) -> MapParser<Self, F, T, U>;
    fn and_then<U, P>(self, other: P) -> AndThenParser<Self, P, T, U>;
    fn or<P>(self, other: P) -> OrParser<Self, P, T>;
    fn many(self) -> ManyParser<Self, T>;
    fn optional(self) -> OptionalParser<Self, T>;
}
```

**设计决策**:
- Parser trait定义组合子操作
- MapParser实现Syntax→Semantic转换
- RecursiveDescentParser构建完整AST

#### 四层转换管道
```rust
pub struct CompilerPipeline {
    target: Target,
}

impl CompilerPipeline {
    pub fn compile(&self, source: &str) -> Result<String, Vec<String>> {
        // Step 1: Syntax Layer - 词法分析
        let tokens = self.lex(source)?;
        // Step 2: Semantic Layer - 语法分析
        let ast = self.parse(&tokens)?;
        // Step 3: Pattern Layer - 类型检查
        self.type_check(&ast)?;
        // Step 4: Domain Layer - 代码生成
        self.generate(&ast)
    }
}
```
