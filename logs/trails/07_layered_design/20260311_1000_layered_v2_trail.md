## Step 1: Web Research 详细记录

### 搜索主题 1: Compiler Architecture Layered Design 2025

**关键发现**:
- [A Layered Certifying Compiler Architecture (2025)](https://webspace.science.uu.nl/~swier004/publications/2025-funarch.pdf) - 最相关
  - 四层架构：Specification → Interface → Automation → Verification
  - 翻译验证而非完整验证
  - 适用于活跃演化的开源编译器

- Modular的Concentric Circle架构（Chris Lattner）
  - "Drop into the layer you need"
  - 零绑定扩展语言

**洞察**: 2025年的研究强调增量、模块化的验证方法，而非整体验证。

### 搜索主题 2: Incremental Compilation Rust 2025

**关键发现**:
- Rust编译器性能调查（2025年9月）：3700+回复
- 增量重建是#1痛点
- 55%开发者等待超过10秒
- "Relink, Don't Rebuild"提案
- Alejandra González的概念重设计：
  - Atomic levels（原子级别）
  - Data dependencies（数据依赖）
  - Two-stage fingerprint verification

**洞察**: 分层架构可以支持增量编译，只重新编译变更的层。

### 搜索主题 3: DSL Design Patterns Implementation

**关键发现**:
- Martin Fowler的DSL模式目录
- 语言利用模式：Piggyback, Specialization, Extension
- 实现模式：词法处理、源到源转换、数据结构表示
- Kotlin DSL模式：Lambdas with Receivers, @DslMarker

**洞察**: DSL实现通常采用多阶段处理管道，与分层架构天然契合。

### 搜索主题 4: Syntax Directed Translation

**关键发现**:
- SDT = Grammar + Semantic Rules
- S-attributed（综合属性）：自底向上
- L-attributed（继承属性）：自顶向下
- 应用：表达式求值、类型检查、中间代码生成

**洞察**: 语法导向翻译是层间转换的理论基础。

### 搜索主题 5: Intermediate Representation Design LLVM MLIR

**关键发现**:
- MLIR设计原则：
  - Parsimony（极简）
  - Traceability（可追溯）
  - Progressivity（渐进性）
- Dialect系统支持领域特定扩展
- 渐进式Lowering避免过早降层

**洞察**: MLIR的渐进式Lowering与我们的分层架构理念一致。

### 搜索主题 6: Gradual Typing Boundaries

**关键发现**:
- 边界是静态类型和动态类型的接口
- Higher-Order Contracts（Typed Racket方法）
- Deep vs Shallow类型（PLDI 2022）
- Wrapper accumulation问题

**洞察**: 渐进类型的边界概念可以直接应用于层间边界检查。

## Step 2: 假设提出

### 技术假设：四层架构如何实现渐进式边界？

**假设**: 使用Rust的trait系统可以实现编译期层间契约验证。

**推理**:
1. 每层暴露精心设计的API
2. 向上层提供保证，向下层隐藏细节
3. 边界检查在层间进行

### 实现假设：Rust中如何实现类型安全的层间转换？

**假设**:
1. 每一层定义一个`Layer` trait
2. 层间转换使用`TryFrom`/`TryInto`
3. 使用`PhantomData`标记层间依赖
4. 通过`#[repr(transparent)]`实现零成本抽象

### 性能假设：分层设计对编译/执行性能的影响？

**假设**:
1. 编译期：实现增量编译
2. 执行期：通过内联消除层间开销
3. 内存：层间传递使用引用

### 适用性假设：哪些场景最适合使用分层架构？

**假设**: 最适合：
1. 领域特定语言（DSL）
2. 安全关键系统
3. 异构计算
4. 长期演进系统

## 关键设计决策记录

### 决策1: 使用PhantomData标记层间关系

**原因**: 在编译期强制层间顺序，防止错误连接。

### 决策2: 每层独立的Context

**原因**: 隔离状态，支持并行和增量编译。

### 决策3: 统一的LayerError类型

**原因**: 简化错误处理，支持跨层错误传播。

### 决策4: 递归下降而非Parser Generator

**原因**:
- 更好的错误信息
- 更容易理解和修改
- 与Rust生态更契合

### 决策5: 简化类型系统（无泛型）

**原因**: 保持代码复杂度可控，专注于层间架构。

## 遇到的问题和解决方案

### 问题1: 左递归处理

**现象**: 表达式解析会导致栈溢出。

**解决**: 使用循环替代递归，实现优先级爬升。

### 问题2: 类型推导中的循环引用

**现象**: 递归函数类型无法推导。

**解决**: 先注册函数签名到符号表，再分析函数体。

### 问题3: 层间错误上下文丢失

**现象**: 错误信息不够精确。

**解决**: 在LayerError中包含层ID和位置信息。

- 编译期类型安全

借鉴渐进类型：
- 静态保证 + 动态检查

分层架构天然支持：
- 每层独立缓存

- 零成本抽象
